Yandex VM SSH
$ ssh-keygen -t ed25519

После выполнения команды ssh-keygen будут созданы два файла — с закрытым и открытым ключом 
(название и расположение файлов можно указать вручную в момент выполнения программы, 
по умолчанию это папка ~/.ssh и файлы id_ed25519). Если вы планируете создавать отдельные ключи для разных серверов или сервисов, 
давайте файлам значимые имена. Например, ключи для подключения к Yandex Cloud можно сохранить в файлах yc_ed25519


Подключение к серверу используя путь до ключа:

$ ssh -i <путь_к_ключу/имя_файла_ключа> <имя_пользователя>@<публичный_IP-адрес_ВМ>


================= Ошибка подключения ==================

@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
@         WARNING: UNPROTECTED PRIVATE KEY FILE!          @
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
Permissions 0755 for '/home/ivan/.ssh/id_ed25519' are too open.
It is required that your private key files are NOT accessible by others.
This private key will be ignored.
Load key "/home/ivan/.ssh/id_ed25519": bad permissions
timofei@158.160.119.68: Permission denied (publickey).


Проблема заключается в том, что права доступа к файлу приватного ключа /home/ivan/.ssh/id_ed25519 слишком широкие, 
что делает его доступным для других пользователей системы. SSH требует, чтобы файлы приватных ключей были защищены 
и не доступны другим пользователям, кроме владельца. В данном случае права доступа установлены как 0755, 
что позволяет другим пользователям читать файл.

Чтобы решить эту проблему, вам необходимо изменить права доступа к файлу, установив их так, 
чтобы только владелец мог читать и записывать файл, а остальные — никаких прав не имели. Это можно сделать с помощью команды 
chmod в терминале. Выполните следующую команду:

chmod 600 /home/ivan/.ssh/id_ed25519

Эта команда устанавливает права доступа к файлу так, что владелец может читать и писать файл (6), 
а группа и другие пользователи не имеют никаких прав (00).

После выполнения этой команды попробуйте снова подключиться к серверу с помощью SSH. Это должно решить проблему 
с предупреждением о незащищенном файле приватного ключа.
