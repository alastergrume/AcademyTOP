1. To start Jenkins, use the following command:

sudo systemctl start jenkins 


2. Similarly, to stop Jenkins, use the following command:

sudo systemctl stop jenkins 

3. To restart Jenkins, use the following command:

sudo systemctl restart jenkins

4.To check the status of the Jenkins service, use the following systemctl command:

sudo systemctl status jenkins

5. You should see the following output:

Включение Jenkins в группу sudo

Чтобы добавить правила в файл sudoers, следует использовать команду visudo, которая автоматически проверит файл на наличие синтаксических ошибок перед сохранением. Использование visudo помогает предотвратить внесение ошибок, которые могут привести к невозможности повышения прав через sudo.

Вот как можно правильно добавить правило в файл sudoers:

Введите в терминале команду sudo visudo. Это откроет файл sudoers в текстовом редакторе (по умолчанию — это может быть nano или vi, в зависимости от дистрибутива).

Добавьте в файл следующую строку:

jenkins ALL=(ALL) NOPASSWD: /bin/cp

Это правило разрешит пользователю jenkins выполнять команду cp от имени всех пользователей без запроса пароля.

Сохраните изменения и закройте редактор. Если вы используете nano, это можно сделать с помощью сочетания клавиш Ctrl + O, затем Enter и Ctrl + X для выхода. Если вы используете vi, введите :wq (сохранить и выйти) и нажмите Enter.

Добавление правила в sudoers через visudo обеспечивает, что все изменения применяются корректно и безопасно. После того как вы добавили правило в sudoers, пользователь jenkins сможет выполнять заданную команду без запроса пароля.
