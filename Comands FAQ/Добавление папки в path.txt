ти предупреждения указывают на то, что скрипты tqdm и isympy были установлены в каталог /var/lib/jenkins/.local/bin, который в данный момент не включен в переменную окружения PATH. Переменная PATH определяет, в каких каталогах операционная система будет искать исполняемые файлы, когда вы запускаете программу из командной строки. Если каталог не входит в PATH, вы не сможете запустить программу, не указав полный путь к ней.

Чтобы решить эту проблему, вам необходимо добавить /var/lib/jenkins/.local/bin в переменную PATH. Как это сделать, зависит от вашей операционной системы и конкретной ситуации, но ниже представлены основные способы:

Для временного решения (только для текущей сессии терминала):
Вы можете временно добавить каталог в PATH с помощью команды export в Linux/Unix:

export PATH=$PATH:/var/lib/jenkins/.local/bin

Для постоянного решения:
Чтобы решение было постоянным, вам нужно добавить эту команду в файл инициализации оболочки, например, .bashrc или .profile, который загружается каждый раз при входе в систему.

Откройте файл .bashrc в домашнем каталоге пользователя jenkins (или другом соответствующем файле инициализации, в зависимости от используемой оболочки и настроек системы) с помощью текстового редактора:

nano /var/lib/jenkins/.bashrc

Добавьте в конец файла следующую строку:

export PATH=$PATH:/var/lib/jenkins/.local/bin

Сохраните и закройте файл. Чтобы изменения вступили в силу, нужно перезагрузить файл инициализации:

source /var/lib/jenkins/.bashrc

или просто перезайдите в систему или перезапустите терминал.

Для пользователей Jenkins:
Если вы настраиваете PATH для Jenkins, возможно, вам потребуется добавить путь в конфигурацию Jenkins, чтобы он был доступен во время выполнения сборок. Это можно сделать через веб-интерфейс Jenkins:

Перейдите к настройкам узла сборки, которые находятся в «Manage Jenkins» > «Manage Nodes and Clouds» > (выберите узел) > «Configure».
В разделе «Node Properties» проверьте «Environment variables».
Нажмите «Add» и добавьте переменную с именем PATH и значением $PATH:/var/lib/jenkins/.local/bin (или альтернативно, добавьте новый путь к существующей переменной через веб-интерфейс Jenkins).
После этого Jenkins будет использовать обновленный PATH в рамках выполнения сборок.
